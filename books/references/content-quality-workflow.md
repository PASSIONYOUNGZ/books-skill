# 内容质量工作流

## 目标

整理稿不是普通摘要，而是“替代第一遍精读”的学习材料。读者读完后，应能说出原书的核心观点、方法步骤、案例逻辑、适用边界和作者的判断方式。

## 三遍工作法

### 1. 内容地图

写正文前，先在工作过程中建立内容地图。内容地图不一定输出，但必须真实完成：

- 原书的章节、附录、练习、案例、对话脚本、Q&A、访谈或参考条目结构。
- 每个章节真正解决的问题。
- 作者反复强调的核心判断。
- 可操作的方法、模型、步骤、检查清单。
- 重要案例、练习、对话、表达脚本、数据、图表、示例，以及它们证明什么。
- 重复内容、广告内容、目录噪声、页码残留和可合并段落。
- 原书没有讲清、抽取不完整或 OCR 质量差的地方。

禁止只看目录或关键词直接写最终正文。

### 2. 学习型整理

按原书真实结构写，而不是按固定模板机械填空。每个重要内容群要让读者学会：

- **是什么**：作者使用这个概念、模型、方法或表达脚本时，到底指什么。
- **为什么**：它解决什么问题，背后的判断逻辑是什么。
- **怎么做**：步骤、顺序、前提、观察信号和失败处理。
- **例子说明**：案例中的情境、动作、对方反应、结果和可迁移结论。
- **边界限制**：什么时候不适用，哪里可能造成误解、误用、操控或伤害。

不同书型采用不同展开方式：

- 教材/理论书：重点保留概念定义、论证链条、模型关系、练习方法。
- 方法书/工具书：重点保留目标、步骤、判断信号、失败处理和复盘方式。
- 对话脚本/表达模板书：不要只堆原话；要说明场景、意图、语言机制、转折点、替代表达和风险边界。
- 案例书/故事书：不要只复述情节；要提炼案例证明的观点和可迁移方法。
- Q&A/帖子合集：按问题群、共识、分歧、典型案例和边界重组。
- 小说/叙事非虚构：保留人物、时间线、冲突、关键场景、叙事视角、主题和情感推进。
- 学术/研究/专业手册：保留问题定义、术语、证据、方法、推理链、限制、争议和应用条件。

### 3. 提炼编辑

写完后必须做一遍编辑，不合格就返工：

- **去重复**：同一观点只在最合适的位置讲透一次。后文需要引用时，只简短回扣，不整段重讲。
- **去啰嗦**：删除“本节围绕”“可以帮助读者”“我们可以看到”等空转句。每段必须提供观点、方法、例子、边界或结构推进。
- **补缺口**：如果章节很短，检查是否漏了方法步骤、案例解释、失败信号或边界；如果原书本身短，明确说明信息密度限制，不编造。
- **保精髓**：保留作者的关键判断和原书特殊表达，不把所有书都改写成同一种通用沟通建议。
- **保比例**：长章节、方法密集章节、案例密集章节要明显展开；过渡章节可以短，但不能空。

## 密度标准

不要用固定字数替代质量，但可以用以下标准判断风险：

- 重要章节通常需要多个自然段，至少讲清观点、方法、例子或边界中的三项。
- 方法、流程或表达脚本章节至少要包含：使用目标、适用场景、操作顺序、反馈信号、失败/风险提醒。
- 案例密集章节至少要包含：情境、做法、结果、作者想证明的结论。
- 总结部分不能只是“本书讲了某主题很重要”，必须抽出全书的主线、方法体系、叙事主题、证据结构、适用人群和限制。

## 反模式

以下情况视为内容质量失败：

- 看起来很长，但大量复制原文、目录、广告、重复段落或模板小标题。
- 每章都写成同样的“核心观点、方法、案例、边界”，但内容没有真实差异。
- 只告诉读者泛泛结论，没有进入原书的具体方法、证据、案例、情节或论证。
- 只罗列表达脚本，不解释为什么这样说、何时说、对象如何反应、哪里不能用。
- 为了显得完整而编造原书没有的案例、模型、观点。
- 为了压缩而删掉原书真正重要的论证、步骤、案例和练习。

## 最终读者测试

交付前问自己：

1. 读者不打开原书，能不能复述作者最重要的观点？
2. 读者能不能照着整理稿理解或练习原书的方法？
3. 读者能不能知道哪些方法、表达脚本或技巧有风险、边界和失败信号？
4. 整理稿有没有明显重复、空话、套话或长而无用的段落？
5. 如果某章很短，是因为原书确实短，还是因为我没有读透？

## 最大自治质量闸门

尽量由代理自己把控质量，减少用户介入：

- **红旗**：默认阻断导出。代理必须回到原文和整理稿定位问题，直接改写、扩写、去重或重组；不能把红旗清单丢给用户判断。
- **警告**：默认进入代理自审。代理需要判断警告是否源于原书本身、抽取质量、书籍体量，还是整理稿质量问题；能修就修，合理保留则在自检中说明。
- **通过**：仍需抽样阅读，不能只因为脚本通过就认定内容合格。
- **质量取舍**：如果“全部处理完”和“内容真正合格”冲突，默认选择内容质量。可以减少本轮处理数量、增加单本精读时间、重写重点书，但要在报告中说明。
- **篇幅取舍**：如果原书信息密度高，允许长；如果整理稿重复啰嗦，必须压缩；如果原书短或抽取残缺，不能编造补长，必须说明限制。

默认不要询问用户这些问题：

- 哪些红旗要不要修。
- 章节应该扩写还是压缩。
- 是否为了质量减少处理数量。
- 重复内容是否删除。
- 常规排版、加粗、结构重组和 EPUB 重建。

遇到硬阻断也默认不中断工作流：

- **源文件缺失**：在 manifest 和目录中复查；找不到就写放弃说明，继续下一本。
- **扫描版或 OCR 极差**：不要硬凑；写明扫描/OCR 限制，继续下一本。
- **依赖缺失或需要联网/安装**：先用现有工具、备用库或降级方案；不能处理就记录环境限制，继续其它可处理书籍。
- **越权写入或权限不足**：改用允许的输出位置；仍不行就记录阻断，继续其它项目。
- **需要删除、覆盖、移动等不可逆操作**：不执行；保留现状，另写新文件或记录待处理项。
- **互斥目标无法同时满足**：按用户长期偏好默认选内容质量；在最终报告说明取舍。
- **敏感或高风险内容边界不清**：按尊重、自愿、可拒绝、非操控的更保守标准处理；在自检中说明。

原则：能继续处理其它书，就继续。不要因为单本书、单个依赖、单个权限问题中断整批任务。

代理自审时优先按这个顺序处理：

1. 修红旗：结构异常、过短、重复过高、工具痕迹、模板短语过多。
2. 修知识缺口：方法、案例、示例表达、练习、边界密度不足。
3. 修表达密度：删空话，合并重复，补必要说明。
4. 修排版：清噪声，加重点，重建 EPUB。
5. 写自检说明：说明哪些警告已解决，哪些因原书体量或抽取限制保留。
