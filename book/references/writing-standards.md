# 写作标准

## 成品结构

每本完成的书，默认使用四个一级标题：

1. 原书的章节结构。
2. 按照章节结构做的详细整理。
3. 本书的整体总结。
4. 打分。

根据书籍类型调整展开方式：

- 正式章节书：尽量沿原章节推进。
- 论文集、帖子合集、文章合集：按真实内容主题和代表性文章组重建结构。
- Q&A 合集：按回答群、重复共识、典型案例、分歧和风险提醒重建结构。
- 小说、传记、回忆录、叙事非虚构：保留人物、时间线、冲突、转折、叙事视角、主题和关键场景，不要改写成纯方法清单。
- 学术、研究、技术、专业手册：保留问题定义、术语、证据、推理链条、实验/数据、限制、应用条件和争议。
- 参考书、词典、资料汇编：按条目体系、主题簇、索引逻辑和高价值条目重建结构，避免逐条机械复述。
- 正文不完整：在整体总结里说明缺失，不要把抽取日志写进正文。

## 详细度要求

每个主要章节或内容群按书型整理。非虚构、教材、方法书通常至少整理：

- 作者要解决的问题。
- 作者的论证或方法。
- 具体步骤、练习、框架或检查清单。
- 重要案例，以及案例证明了什么。
- 风险、限制、失败情形。
- 本章和下一章的递进关系。

叙事类作品至少整理人物关系、情节推进、冲突变化、关键场景、叙事手法和主题含义。学术/专业类作品至少整理核心问题、概念定义、证据来源、推理链、限制和应用条件。

不要把章节压缩成关键词列表。

## 内容密度要求

质量不是越长越好，也不是越短越好。整理稿必须做到“有信息密度的忠实展开”：

- **长而不散**：长章节可以展开，但不能反复复制同一观点、同一案例、同一套小标题。重复内容要合并，保留最能代表原书思想的位置。
- **短而不空**：短章节可以简洁，但必须说明它在原书中的作用；如果涉及方法、案例、脚本或示例表达，不能只写结论。
- **段落有功能**：每个自然段都要承担一个功能：解释观点、定义概念、说明方法、拆解案例、提示边界、连接章节递进。没有功能的过渡句要删除。
- **比例要合理**：原书重点章节、方法密集章节、案例密集章节应明显更详细；过渡、广告、重复目录、低价值段落应压缩或删去。
- **不要模板化平均用力**：不是每章都必须同样长，也不是每章都机械写“观点、方法、案例、边界”。按原书真实材料决定展开方式。

## 知识吸收标准

整理稿的目标不是“告诉读者这本书大概讲什么”，而是让读者在不打开原书的情况下，尽可能吸收到原书的知识、观点、思想、方法和判断方式。写作时必须做到：

- **讲清观点**：说明作者真正主张什么，反对什么，为什么这样主张。
- **讲清方法**：如果原书给了步骤、脚本、练习、模型或检查清单，整理稿要把用法、适用场景、操作顺序和失败信号讲清楚。
- **讲清例子**：重要案例不能只说“举了案例”，要说明案例中的情境、做法、结果，以及它证明了什么。
- **讲清递进**：章节之间如果有从概念到方法、从开场到推进、从观察到反馈、从问题到修复的关系，要把递进关系写出来。
- **讲清限制**：尤其是医疗、法律、金融、心理判断、亲密关系、高风险说服、政治宣传、商业承诺等可能影响他人权益的内容，必须区分作者观点、可学习方法、证据强度和需要警惕的误用风险。
- **保留原书质感**：原书如果是教材，就保留概念框架和论证；如果是脚本/表达模板书，就重建场景、意图、使用边界和替代表达；如果是访谈或讲义，就保留问答脉络、反复强调的原则和现场修正；如果是小说或回忆录，就保留叙事节奏、人物弧线、关键场景和主题张力。

合格的章节整理通常需要多个自然段，而不是一两句结论。短书可以短一些，但不能空；长书、教材、方法书、案例密集书必须明显展开。任何“关键词 + 模板句”的结果都视为不合格。

## 方法、案例、脚本写法

遇到方法、模型、练习、案例、对话脚本和表达模板时，按内容本身选择展开角度：

- **方法/模型**：讲清目标、适用前提、操作顺序、观察信号、失败处理、限制。
- **案例/故事**：讲清情境、做法、参与者反应或结果、作者用它证明的观点；叙事作品还要讲清人物动机、冲突变化和主题功能。
- **对话脚本/表达模板**：讲清场景、意图、转折点、语言机制、可替代表达、风险边界；不要只堆原句。
- **练习/清单**：讲清练习目的、执行步骤、检查标准、常见错误。
- **概念/理论**：讲清作者对该概念的用法，不要只给通用百科式定义。

## 排版

- 四个顶层部分使用 `#`，内部使用 `##` 和 `###`。
- 对核心概念、模型名、关键方法、关键判断句适度加粗。
- 不要整段加粗，也不要每句话都加粗。
- 步骤、练习、框架、复习点用列表。
- 主要解释仍以自然段为主，避免全篇碎片化。
- 不保留目录页点线、页码引导符、行首页码、广告尾巴和 OCR 噪声。`………………………………………………………………………`、`....................`、`__________`、`- 25 -`、连续横线这类内容通常必须删除或压缩。
- 普通语气停顿最多保留 `……`。不要让连续省略号、连续点号或连续下划线成为视觉主体。
- 每篇整理稿要有明显的阅读抓手。核心框架、方法名、模型名、风险提醒、适用边界、关键判断句应有适度加粗，让读者扫读时能抓住重点。
- 如果原书是脚本库或表达模板集，除正文解释外，应尽量把脚本的使用意图、适用场景、转折点、失败信号和边界提醒标清楚，而不是只堆原句。

## 禁止模式

避免出现：

- “本节围绕……”
- “主题线索……”
- “来源统计……”
- “OCR 状态……”
- “关键词集中在……”
- “内容归入……”
- “可以帮助读者……”
- “有助于我们理解……”
- “这一章主要讲了……”，但后面没有具体观点、方法或例子。
- 每章只写固定句式，如“先看场景、再选择表达、最后根据反馈修正”。
- 原始文件名、哈希、页抽取日志、工具运行痕迹。
- 每章重复同一套模板句。
- 半原文半摘要的拼贴段落。
- 没有原书具体观点、方法、例子、情节或证据，只是在讲通用领域知识。
- 长串省略号、点线、下划线、横线直接出现在正文里，造成“未清洗抽取文本”的观感。
- Markdown 从头到尾几乎没有加粗、重点标记或清晰的内部小标题。
- 长篇段落看似丰富，实则不断重复同一观点、同一原文片段或同一组泛泛提醒。
- 章节过短，缺少原书的重要方法、案例、练习、示例表达或判断标准。

## 打分

按 10 分制评分，简要说明：

- 内容完整度。
- 原书结构清晰度。
- 忠实原文程度。
- 实操、审美、研究或理解价值。
- 案例、方法、练习密度。
- 抽取质量或源文件限制。
